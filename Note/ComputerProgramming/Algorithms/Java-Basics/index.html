    Java基础原创

    # Java 基础

    Java 是一个完全面向对象的语言，提供了一系列数据抽象和封装的工具方法，这一点和算法设计与应用的理念不谋而合。

    我们将算法拆分成实现层和应用层两个层面，实现层专注于算法逻辑和思想，提供一层通用的 API(Application Programing Interface, 应用程序编程接口)，而应用层关注将具体问题进行算法建模，从而可以应用一些设计优秀的算法去解决实际问题。

    而Java语言正式利用面向对象的思想进行模块化编程的，可以完美符合算法设计需求。

    简而言之，Java 面向对象增强了模块化编程，将实现和用例完全拆分，这样的特性可以帮助我们更加清晰地进行算法设计和应用。

    # 1. Java 基础编程模型

    为了避免算法依赖特定的编程语言，因此这里只用了Java语言的一个子集，优先使用大部分现代编程语言所共有的语法。

    # 1.1 Java 基本介绍

    建议读者在阅读前最好掌握一门基本语言，如： C/C++、Python 等语言。
    因为本专栏着眼于算法而非语言，所以这里只会简单介绍Java，语言之间的通用语法本文不进行赘述，只进行知识逻辑的铺排。

    一段 Java 程序（类）一般是一个静态方法（函数）库或定义了一个数据类型。 有以下七种基本语法：

    • 原始数据类型与表达式：
      int, double, boolean, char, long, short, byte, float.

      使用 中缀表达式 ，基本运算符大致分为 算术运算符，逻辑运算符 和 比较运算符（布尔表达式），支持类型转换（自动转换下自动提高数据类型）。

    • 语句：创建变量并对其进行赋值、控制流程，或引发副作用来计算。

      六种基本语句类型：声明、赋值、条件、循环、调用和返回。

      副作用：方法 除了计算和返回值外所做的任何事情都是副作用 。
      例如：在屏幕上绘制某些内容，写入文件或者网络连接。

    • 静态方法：可以封装和重用代码，进行模块化编程。

    • 标准输入输出：程序与外界联系的桥梁。

      Princeton University为我们提供了一系列的静态方法库方便我们进行代码的编写，其API已在后面小节给出。

    • 数据抽象：定义非原始数据类型，面向对象编程。

      对象 = 引用类型 + 一系列操作

    • 数组：同一类型多个值的集合，是一个对象。

    • 字符串：一连串的字符，是一个对象。

    关于数据抽象和面向对象编程我们将在下一节讨论。

    由于数组和字符串也是对象（Java 内置的），因此和引用类型一同在下一节讨论。

    下面阐述 基础编程模型——模块化编程。

    # 1.2 模块化编程

    • 静态方法库是定义在一个 Java 类中的一组静态方法。
    • 类的声明是 public class 加上类名，以及用花括号包含的静态方法。
    • 存放类的文件的文件名和类名相同，扩展名是 .java。
    • Java 开发的基本模式是编写一个静态方法库（包含一个 main() 方法）来完成一个任务。
    • 输入类名以及一系列字符串就能调用类中的 main() 方法，其参数为由输入的字符串组成的一个数组。
    • main() 的最后一条语句执行完毕之后程序终止。

    当我们提到用于执行一项任务的 Java 程序时，我们指的是用这种模式开发的代码（可能还包括对数据类型的定义）。

    这个模型最重要的一点就是利用静态方法库实现了模块化编程。

    这样做有很多好处：

    1. 将一个复杂的问题拆分为各个模块（静态方法库）逐个击破；
    2. 缩小测试范围——单元测试；
    3. 可以共享和重用代码来避免对一个算法的重复实现；
    4. 很容易用改进过的实现替换老的实现。

    # 1.3 单元测试

    Java 支持在每个静态方法库中都包含一个 main() 方法。
    我们可以将一个静态函数库中的 main() 方法作为一个 开发用例，以便于用其 测试每一个小模块 的正确性。

    main() 方法还可作为 测试用例，对 全部代码进行测试（应用），用例复杂时，我们会将其独立为一个模块（静态方法库）。

    # 1.4 API

    模块化编程的一个重要组成部分就是 记录库方法的用法并供其他人参考的文档。

    我们会统一使用 应用程序编程接口（Application Programing Interface）的方式列出使用的每个库方法名称、签名和简短的描述。

    我们用 用例 来指代调用另一个库中的方法的程序，用 实现 描述实现了某个 API 方法的 Java 代码。

    我们主要使用4种不同类型库中的静态方法：
    系统标准库、导入的系统库、Princeton标准库 和自己编写的库（如 BinarySearch.java）。
    这些常用库的 API 将在本章最后一节列出。
